# OA Review macOS 文件说明

这个文件夹只整理了 macOS 适配相关脚本和必要源码，不包含 A1689 的图片、CSV、PDF 等数据。

## 文件结构

- `build_macos.sh`: 在 Mac 上生成 `OA_Review.app`。
- `OA_Review_macos.spec`: PyInstaller 的 macOS 打包配置。
- `requirements-oa-review.txt`: macOS Python 依赖。
- `oa_review_launcher.py`: 打包后的程序入口。
- `oa_review_gui.py`: OA Review 主程序源码。
- `A1689/OA_Review.command`: 不打包时，双击/打开运行 A1689 的脚本。
- `A1689/visualisation.py`: A1689 的 Python 启动包装脚本。
- `README_macos.md`: 英文版简要说明。

## 推荐用法：放回完整项目后打包

在 Mac 上保留完整项目结构，例如：

```text
Astronaut_OA/
  oa_review_gui.py
  oa_review_launcher.py
  build_macos.sh
  OA_Review_macos.spec
  requirements-oa-review.txt
  A1689/
    vis_table.csv
    annotation_check_triplets/
    visualisation.py
    OA_Review.command
```

然后在 Mac 终端运行：

```bash
cd /path/to/Astronaut_OA
bash build_macos.sh A1689
open A1689/OA_Review.app
```

## 不打包直接运行

```bash
cd /path/to/Astronaut_OA
python3 -m venv .venv-oa-review
source .venv-oa-review/bin/activate
python -m pip install -r requirements-oa-review.txt
chmod +x A1689/OA_Review.command
open A1689/OA_Review.command
```

注意：`OA_Review.exe` 不能直接在 macOS 原生运行，必须在 Mac 上用这些源码和脚本重新运行或重新打包。

## 没有 Mac 时的远程打包

可以使用 GitHub Actions 的 macOS runner 自动打包。相关文件已放在：

```text
.github/workflows/build-macos.yml
GitHub_Actions_macOS打包说明.md
```

上传到 GitHub 前不要上传 `.venv-win-verify`、`dist-win-verify` 这些 Windows 验证目录；本目录里的 `.gitignore` 已经排除了它们。
